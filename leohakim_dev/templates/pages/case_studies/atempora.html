{% if LANGUAGE_CODE == 'en' %} Product architecture case {% else %} Caso de arquitectura de producto {% endif %}

Atempora

{% if LANGUAGE_CODE == 'en' %} A product architecture case for a SaaS with scheduling, credits, payments, and complex rules that could not be improvised without creating expensive debt later. {% else %} Un caso de arquitectura de producto para un SaaS con agenda, créditos, pagos y reglas complejas que no se podían improvisar sin generar deuda cara después. {% endif %}

{% if LANGUAGE_CODE == 'en' %} Context {% else %} Contexto {% endif %}

{% if LANGUAGE_CODE == 'en' %} The product combines sessions, credits, payments, cancellations, currencies, settlements, and multi-tenant rules. The risk was ending up with an MVP that looked functional on the surface but was inconsistent underneath. {% else %} El producto combina sesiones, créditos, pagos, cancelaciones, monedas, liquidaciones y reglas multi-tenant. El riesgo era terminar con un MVP que pareciera funcional por fuera pero fuera inconsistente por dentro. {% endif %}

{% if LANGUAGE_CODE == 'en' %} My role {% else %} Mi rol {% endif %}

{% if LANGUAGE_CODE == 'en' %} As founder and backend/product architect, I focused on designing the domain, the API, and the core rules before the product scaled on top of ambiguous assumptions. {% else %} Como fundador y arquitecto de backend/producto, me centré en diseñar el dominio, la API y las reglas centrales antes de que el producto creciera sobre supuestos ambiguos. {% endif %}

{% if LANGUAGE_CODE == 'en' %} Approach {% else %} Enfoque {% endif %}

  1. {% if LANGUAGE_CODE == 'en' %} 1. Model business rules before screens: {% else %} 1. Modelar reglas de negocio antes que pantallas: {% endif %} {% if LANGUAGE_CODE == 'en' %} The first job was to define what sessions, credits, cancellations, usage rules, and settlements really meant instead of letting the implementation guess. {% else %} El primer trabajo fue definir qué significaban realmente sesiones, créditos, cancelaciones, reglas de uso y liquidaciones en lugar de dejar que la implementación lo adivinara. {% endif %}
  2. {% if LANGUAGE_CODE == 'en' %} 2. Separate key concepts on purpose: {% else %} 2. Separar conceptos clave a propósito: {% endif %} {% if LANGUAGE_CODE == 'en' %} Credits were decoupled from money, sessions and slots were modeled explicitly, and multi-tenant plus settlement logic were treated as first-class concerns instead of future patches. {% else %} Los créditos se desacoplaron del dinero, sesiones y slots se modelaron de forma explícita, y la lógica multi-tenant y de liquidaciones se trató como algo central y no como un parche futuro. {% endif %}
  3. {% if LANGUAGE_CODE == 'en' %} 3. Build the API around real product constraints: {% else %} 3. Construir la API alrededor de restricciones reales del producto: {% endif %} {% if LANGUAGE_CODE == 'en' %} The goal was not just cleaner code, but a backend structure the product could keep extending without contradictions becoming structural debt. {% else %} El objetivo no era solo un código más limpio, sino una estructura backend que el producto pudiera seguir extendiendo sin que las contradicciones se convirtieran en deuda estructural. {% endif %}

{% if LANGUAGE_CODE == 'en' %} Result {% else %} Resultado {% endif %}

{% if LANGUAGE_CODE == 'en' %} The most important result is clarity. The product now has a much more coherent base for implementation, with fewer dangerous ambiguities in the core rules that matter most. {% else %} El resultado más importante es la claridad. El producto ahora tiene una base mucho más coherente para implementarse, con menos ambigüedades peligrosas en las reglas centrales que más importan. {% endif %}

{% if LANGUAGE_CODE == 'en' %} Business value {% else %} Valor de negocio {% endif %}

{% if LANGUAGE_CODE == 'en' %} In a product like this, good architecture is business value. It reduces the chance of expensive rewrites, inconsistent behavior, and avoidable friction once the MVP starts growing. {% else %} En un producto así, una buena arquitectura ya es valor de negocio. Reduce la posibilidad de reescrituras caras, comportamientos inconsistentes y fricción evitable cuando el MVP empiece a crecer. {% endif %}

{% if LANGUAGE_CODE == 'en' %} Simple stack {% else %} Stack simple {% endif %}

{% if LANGUAGE_CODE == 'en' %} Django and DRF for the backend and API, PostgreSQL plus queues for the core logic, React for the product interface, and payment/video providers connected to the domain rather than driving it. {% else %} Django y DRF para backend y API, PostgreSQL y colas para la lógica central, React para la interfaz de producto, y proveedores de pago/video conectados al dominio en lugar de definirlo. {% endif %}

{% if LANGUAGE_CODE == 'en' %} This is presented as a product architecture case and an example of capability, not as a finished commercial success story. {% else %} Esto se presenta como un caso de arquitectura de producto y una prueba de capacidad, no como una success story comercial ya cerrada. {% endif %}
